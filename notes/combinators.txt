Descendant combinator - represented by a single space ( ) character
combines two selectors such that elements matched by the second selector
are selected if they have an ancestor element matching the first selector.

.box p {
    color: red;
}
This will only select p elements that are descendants of an element with the
"box" class value.

Child combinator - is represented by (>) placed between two CSS selectors.
It matches only those elements matched by the second selector that are the
direct children of elements matched by the first. Descendant elements
further down the hierarchy don't match. 
- E.g. to select only <p> elements that are direct children of <article> elements.

ul > li {
  border-top: 5px solid red;
}
This will only select li(second) element(s) that are direct children of the ul
(first) element(s).

Next-sibling combinator - is represented by a (+) place between two CSS selectors.
It matches only those elements matched by the second element that come right after
the element matched by the first selector. For example, to select all <img> elements
that are immediately preceded by a <p> element.

p + img {
  /* … */
}
This combinator is commonly used to do something with a paragraph that follows a
heading.

Subsequent-sibling combinator - represented by a (~) placed between two CSS selectors.
It is used to select siblings of an element even if they are not directly adjacent.

body {
  font-family: sans-serif;
}

h1 ~ p {
  font-weight: bold;
  background-color: #333333;
  color: white;
  padding: 0.5em;
}
This selects all <p> elements that come after the <h1> element. Even if there is
an element between <p> elements that come after the <h1> element, they will still
be selected.

Combining combinators with selectors - selectors can be combined to select certain
parts of your document. You must be careful when creating big lists of selectors to 
select very specifc parts of your document. It will get hard to reuse the CSS rules
since you have made the selector very specific to the location of that element in the
markup. Instead just use a simple class and apply that to the element in question.
